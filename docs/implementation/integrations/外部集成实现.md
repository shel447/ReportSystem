# 外部集成实现

本篇只记录非 Office Exporter 的外部依赖。Java 文档导出实现已独立进入 [文档导出实现](../document-export/README.md)。

## 1. LLM 集成

LLM 能力只通过统一网关使用，应用层依赖能力 port：

- 模板匹配补充判断
- 参数提取
- 诉求实例化文案整理
- 章节内容生成
- 章节总结和报告汇总

这些能力属于报告生成场景，由 [报告生成实现](../report-generation/README.md) 编排。

## 2. 动态参数数据源

模板只声明 `source`。系统负责：

1. 将当前可见参数组装成标准请求体。
2. 向外部接口发送 `POST`。
3. 校验外部响应满足 `parameter-option-source-response.schema.json`。
4. 返回 `options + defaultValue`。

## 3. 平台外部能力

外部能力按职责拆分 adapter，不使用单个万能 HTTP gateway：

- Guardrail：问题、答案、SQL/Python 安全检查。
- AgentCore：会话、轮次、历史查询和 `chat/import` upsert。
- DataCatalog：逻辑实体、数据集和实体关系。
- Knowledge：知识查询、语义检索和多索引检索。
- OneQuery：执行 SQL/UQL。
- NodeAgent：启动与周期加载平台配置。
- Audit：非阻塞发送操作审计和安全审计。
- Metadata Sync：周期检查元数据版本并触发缓存失效。

底层共享 HTTP client 统一处理 base URL、身份头、认证头、超时和错误转换。业务层不得直接理解外部错误码或布尔语义。重试策略由具体 adapter 按接口幂等语义单独决定，不能在通用 POST client 中隐式开启。

## 4. 查询与执行

数据查询基础设施负责：

- 解释 `runtimeContext.bindings`。
- 运行 SQL 或 API 查询。
- 返回结构化结果和执行证据。

运行时禁止提前把多值参数拼成展示文本再送执行层；执行层消费三元组数组和最终 `resolvedQuery`。

生产 OneQuery 固定路径为 `/rest/dte/v1/onequery/uql/query`。报告 `sql/api` 数据集与智能问数复用 `data_analysis` 中的查询执行和元数据映射能力。

## 5. 运行配置、缓存和调度

- NodeAgent 配置作为生产默认值，环境变量可覆盖，开发设置只用于本地联调。
- DataCatalog 默认缓存 600 秒；Knowledge 与 NodeAgent 使用可替换 TTL cache port 的内存实现。
- APScheduler 在应用生命周期内周期刷新 NodeAgent 配置和元数据版本。
- 元数据更新后清理 DataCatalog、Knowledge 和模板语义索引缓存。
- 审计使用内存队列异步尽力发送；失败记录日志，不阻塞主业务。
