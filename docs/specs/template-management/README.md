# 模板管理

模板管理用于维护可重复使用的报告模板。报告模板描述报告需要收集哪些条件、包含哪些目录或页面、每个章节希望回答什么问题，以及适合展示哪些内容。

## 适用场景

- 为周期性日报、周报、月报建立固定模板。
- 为不同部门维护不同口径的经营分析报告。
- 为演示汇报准备按页面组织的 PPT 模板。
- 从已有材料中整理模板草稿，再由业务人员确认后保存。

## 用户可以做什么

- 查看模板列表和模板详情。
- 创建、编辑和删除模板。
- 导入模板草稿并预览整理结果；连续阅读型和分页汇报型模板都可以通过导入进入系统。
- 导出已有模板，供备份或复用。
- 为模板配置报告分类、名称、说明和生成结构。

## 两种模板结构

| 类型 | 使用方式 | 典型产物 |
|---|---|---|
| 连续阅读型 | 按目录、子目录和章节组织内容，适合从上到下阅读 | Word、长报告页面 |
| 分页汇报型 | 按章节和页面组织内容，每页承载一组重点信息 | PPT、会议汇报 |

## 条件与大纲

- 模板可以要求用户填写日期、区域、门店、指标范围等生成条件。
- 条件可以是固定选项，也可以从外部数据源动态获取候选项。
- 目录标题和章节要求可以引用用户已经选择的条件。
- 章节是内容生成单元，不等同于目录标题。
- 模板可以定义文本、表格、图表和组合表格等展示方式。

## 动态内容

模板可以根据用户选择动态展开不同目录、章节或页面。例如，用户选择三个门店后，可以为每个门店生成一组相同结构的分析内容。

部分目录、章节或页面也可以交由外部能力生成，以便接入已有分析系统。

## 使用限制

- 模板是平台共享配置资产，不属于某个用户的私有数据。
- 模板管理权限由平台统一控制；普通用户是否可以创建、修改或删除模板取决于平台授权。
- 模板保存前必须通过结构校验。
- 同一模板中的关键标识不能重复。
- 模板只描述可复用结构，不保存某次报告生成过程中的生成草稿或临时状态。
- 模板导入预览只帮助用户检查草稿，不会自动保存为正式模板。
- 模板字段和结构受系统固定的模板结构定义约束；业务用户维护的是具体模板，不是系统 Schema。
- 分页汇报型模板当前可以导入、保存、查看结构并参与后续报告生成；页面内细粒度编辑能力会逐步补齐。

字段级配置方式见 [模板技术手册](../../implementation/contracts/manuals/报告模板定义与使用说明书.md)。
