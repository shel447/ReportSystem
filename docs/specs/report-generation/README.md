# 报告生成

报告生成让用户通过对话完成一份正式报告。系统会理解用户想要的报告类型，引导用户补充必要条件，给出可确认的大纲，并在生成过程中持续展示进度和内容预览。

报告生成是通用对话支持的一种业务场景。用户可以直接描述报告需求进入该场景；进入报告场景后，后续参数补充和确认会继续沿用同一流程。

## 典型流程

1. 用户说明希望生成什么报告，例如“生成本月网络运行分析报告”。
2. 系统识别适合的报告模板。
3. 如果条件不完整，系统继续询问日期、区域、范围等必要信息。
4. 系统根据模板和已确认条件生成报告大纲。
5. 用户检查大纲，可以调整章节诉求和重点。
6. 用户确认后，系统逐步生成报告内容。
7. 生成完成后，用户可以预览正式报告，并继续生成 Word、PPT 或 PDF 文档。

## 条件补充

- 系统只询问当前报告真正需要的条件。
- 用户可以一次填写多个条件，也可以分多轮补充。
- 对于门店、区域等动态选项，系统可以加载当前可用候选项。
- 用户确认生成前，可以检查已经生效的条件。
- 参数提取、缺失条件判断、参数追问内容和确认状态都属于报告生成规则；通用对话只负责把追问展示给用户，并把答复送回报告场景。

## 大纲确认

- 大纲按目录、子目录和章节展示报告结构。
- 章节描述该部分希望回答的问题和分析重点。
- 用户可以修改章节诉求，再确认生成。
- 连续阅读型报告按目录组织；分页汇报型报告按章节和页面组织。

## 生成过程

- 系统会持续展示当前阶段和正在生成的部分。
- 页面可以逐步预览已经生成的目录、章节、文本、表格和图表。
- 生成完成后，正式报告进入报告中心，便于后续查看和导出。
- 页面预览与本地编辑使用 BI Engine，确保不同入口看到一致内容。

## 重新生成单个章节

报告完成后，用户可以只调整某个章节的大纲并重新生成该章节：

1. 在报告中选择目标章节。
2. 修改该章节的分析重点。
3. 发起章节重新生成。
4. 预览新章节内容。
5. 决定是否在后续保存流程中采用新内容。

当前章节重新生成只返回预览结果，不会自动覆盖原报告，也不会让已经生成的下载文档失效。

## 动态与外部内容

- 模板可以根据用户选择展开重复结构，例如按多个门店分别生成分析章节。
- 模板也可以在生成过程中接入外部分析能力，获取目录、章节或页面内容。
- 外部内容失败时，系统应给出可定位的错误提示，避免静默生成不完整报告。

## 使用限制

- 用户确认前形成的内容属于生成草稿，不是正式报告。
- 分页汇报型报告的部分自动生成功能仍在逐步完善。
- 报告生成依赖模板质量和可用数据；必要数据缺失时，系统需要明确提示。

内部运行规则、技术契约和字段映射见 [报告生成实现](../../implementation/report-generation/README.md)。
