# 通用对话

通用对话用于承载用户与系统之间的连续交流。报告生成是其中一种业务场景，但普通会话本身不依赖报告模板。

## 用户可以做什么

- 创建新的会话。
- 查看历史会话和消息记录。
- 删除不再需要的会话。
- 从某条历史消息创建分支，在保留原会话的同时探索另一种方向。
- 在系统处理较长任务时查看持续更新的进度。

## 对话体验

- 用户发送问题后，可以逐步看到系统正在处理的阶段。
- 系统需要补充信息时，会明确提出问题并等待用户回答。
- 已经完成的结果会保留在对应会话中，便于后续追溯。
- 会话记录按当前用户隔离，不会与其他用户混在一起。

## 场景扩展

不同业务能力可以复用通用对话。例如报告生成会在对话中引导用户补充条件、确认大纲，并逐步展示报告内容。

报告场景的完整操作方式见 [报告生成](../report-generation/README.md)。
