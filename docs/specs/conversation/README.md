# 通用对话

通用对话用于承载用户与系统之间的连续交流。报告生成是其中一种业务场景，但普通会话本身不依赖报告模板。

## 用户可以做什么

- 创建新的会话。
- 查看历史会话和消息记录。
- 删除不再需要的会话。
- 从某条历史消息创建分支，在保留原会话的同时探索另一种方向。
- 在系统处理较长任务时查看持续更新的进度。

## 对话体验

- 用户发送问题后，可以逐步看到系统正在处理的阶段。
- 用户通常只需要直接描述需求；系统会识别应该进入的业务场景。
- 当系统无法可靠判断用户意图时，会先请用户补充说明，而不是擅自进入某个业务流程。
- 系统需要澄清意图或补充信息时，会明确提出问题并等待用户回答。
- 用户回答后，系统会将答复关联到对应追问，再继续推进当前业务场景。
- 同一段连续交流会优先延续已经进入的业务场景，避免补充信息时意外切换流程。
- 已经完成的结果会保留在对应会话中，便于后续追溯。
- 会话记录按当前用户隔离，不会与其他用户混在一起。

## 场景扩展

不同业务能力可以复用通用对话。例如报告生成会在对话中引导用户补充条件、确认大纲，并逐步展示报告内容。

通用对话只负责承载追问和答复。具体询问什么、答复中包含哪些业务信息，由对应业务场景决定。例如报告生成会询问报告参数；未来其他场景可以使用完全不同的澄清内容。

系统内部也可以明确指定本轮希望进入的业务能力。未明确指定时，通用对话会根据用户输入和前序交流识别适合的场景；识别结果不够明确时，会返回通用澄清问题。

报告场景的完整操作方式见 [报告生成](../report-generation/README.md)。
