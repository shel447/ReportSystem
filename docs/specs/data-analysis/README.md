# 智能问数

智能问数让用户直接用自然语言查询业务数据。系统会理解问题、查询可用数据，并返回结论、明细和适合的可视化结果。

## 用户可以做什么

- 询问指标、趋势、分布和明细，例如“本月核心设备告警按级别怎么分布？”
- 查看系统生成的自然语言结论。
- 查看本次分析的业务标题和查询口径说明。
- 查看本次查询使用的数据和查询语句。
- 查看表格和图表形式的结果。
- 在同一会话中继续追问。
- 用户入口始终是自然语言提问；第三方系统如需从 SQL 等结构化输入接入，属于后续集成能力，不作为当前用户可见入口。

## 数据理解

- 数据目录、知识检索和查询执行都使用当前用户身份，确保结果只来自该用户有权访问的数据范围。
- 系统会结合可用数据目录理解实体、字段和关联关系。
- 系统可以检索指标解释和历史查询样例，提升查询准确度。
- 查询执行前会检查生成语句的安全性。
- 查询结果会保留字段含义和必要的数据血缘，便于追溯。

## 分析过程

一次智能问数会依次完成问题理解、查询生成、数据获取，并在得到数据后并行形成可视化建议和分析结论。

- 用户始终可以从自然语言问题开始分析。
- 外部系统后续也可以从已经准备好的查询语句开始复用数据获取、可视化和总结能力。
- 可视化无法形成合适图表时，系统仍会返回表格结果。
- 查询结果、图表和结论来自同一次数据查询，避免不同展示之间的数据口径不一致。
- 系统会根据问题、字段含义和实际数据选择图表；图表标题、业务结论和查询口径说明保持一致。

## 异常处理

- 用户输入、返回答案或生成的查询存在安全风险时，系统会拒绝继续处理并给出提示。
- 查询服务失败、超时或返回非法数据时，系统会明确提示失败，不把空结果伪装成正常答案。
- 查询使用当前数据源不支持的 `CONNECT BY` 语法时，系统会明确提示该语法不受支持，不自动重复执行同一查询。
- 查询引用的字段不存在时，系统会明确提示字段不存在，不继续生成图表或分析结论。
- 知识检索不可用时，系统仍会尝试完成查询，但结果可能缺少补充解释。

## 与报告生成的关系

智能问数和报告生成共享数据查询、字段解释和可视化能力。智能问数面向即时分析；报告生成面向按模板冻结的正式内容。
