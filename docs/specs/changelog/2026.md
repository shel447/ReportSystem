# 2026 年规格变更日志

本文件只记录用户可感知的功能变化。接口、Schema、代码结构、数据库和 exporter 实现调整统一记录在 [实现变更日志](../../implementation/changelog/2026.md)。

## 2026-06-03 对话运行中协作体验

- 对话中的长流程可以实时返回处理阶段、局部结果、最终结果和错误。
- 长流程可以展示外部工具或查询处理中状态，帮助用户理解等待原因。
- 用户可以取消仍在运行中的任务；需要补充输入的流程可以暂停等待用户继续提供信息。
- 系统可以在无法继续或触发安全限制时主动停止任务或拒绝回答，并说明原因。
- 报告生成接入该机制，生成过程中可持续展示阶段和报告增量。

## 2026-06-03 结构定义与业务实例术语收口

- 业务规格明确“定义”指系统固定的 Schema 或结构定义，不是业务用户维护的具体资源。
- 模板结构定义约束报告模板；报告模板是业务人员维护的具体配置资源。
- 报告结构定义约束正式报告；正式报告是报告中心保存和管理的资源，可以包含部分内容或完整内容。

## 2026-06-02 外部系统接续报告生成

- 外部业务系统已经识别出报告模板并提取出部分全局条件时，可以将结果交给 ReportSystem 继续完成参数补齐、确认和报告生成。
- 外部系统可以同时交接全局条件的候选项、默认值和当前取值，减少重复查询。
- 首版只支持模板根级全局条件；目录、章节和分页页面中的局部条件仍由 ReportSystem 自身流程处理。

## 2026-06-01 用户身份边界与数据隔离收口

- 用户身份由平台统一管理，不在报告系统内维护用户资料。
- 模板是平台共享配置资产；会话、报告草稿、正式报告和导出文档按当前用户隔离。
- 智能问数的数据目录、知识检索和查询执行使用当前用户身份，避免跨用户复用数据。

## 2026-06-01 生产外部能力与智能问数

- 新增智能问数场景：用户可以用自然语言查询数据，并获得结论、明细和可视化结果。
- 通用对话历史改由平台能力托管；当前暂不开放删除历史会话和历史分支。
- 用户问题、系统答案和生成查询在执行前后增加安全检查。
- 报告生成与智能问数共享数据查询、字段解释和可视化能力。

## 2026-06-01 报告查询业务失败降级

- 外部查询返回业务失败时，报告生成允许使用空数据继续完成。
- 系统必须保留可定位告警，避免用户把缺失数据误解为真实的空结果。
- 外部调用超时、HTTP 失败或非法响应仍返回明确错误。

## 2026-06-01 分页模板导入闭环

- 模板导入支持连续阅读型和分页汇报型两类报告模板。
- 分页汇报型模板导入后可以查看结构摘要、保存为正式模板，并用于后续报告生成。
- 分页模板的页面内细粒度编辑能力暂未开放，当前以导入、查看和保存为主。

## 2026-06-01 报告生成外部数据能力收口

- 报告模板可以通过外部动态参数候选项辅助用户选择生成条件。
- 报告生成可以执行 SQL 查询或 API 查询，并将结果用于文本、表格和图表。
- 模板可以继续调用外部动态内容服务生成目录、章节、分页页面或页面组件。
- 外部服务超时、失败或返回非法内容时，系统必须返回明确、可定位的错误。

## 2026-06-01 通用对话场景识别

- 用户未明确指定操作类型时，可以直接描述需求，由通用对话识别适合的业务场景。
- 连续交流优先延续上一轮正在进行的场景；用户回复系统追问时，答复会继续交给原场景处理。
- 无法可靠识别用户意图时，系统先返回通用澄清问题，不再默认进入报告生成。

## 2026-05-31 文档导出可用格式收口

- Word、PPT 和 Markdown 是当前可用的报告导出格式。
- PDF 暂未开放；请求 PDF 时返回明确错误，不生成占位文件。

## 2026-05-31 通用追问与报告参数边界收口

- 通用对话明确提供可复用的追问和答复机制。
- 报告生成继续在对话中引导用户补充参数，但参数内容、缺失判断和确认规则归属于报告业务。
- 后续其他业务场景可以复用多轮澄清体验，而不需要使用报告参数模型。

## 2026-05-31 业务规格与技术实现边界收敛

- `docs/specs/` 改为面向产品人员和业务用户的功能说明。
- API、Schema、示例和字段级手册迁入实现设计中的技术契约区。
- 文档导出规格新增用户可感知配置能力表，区分当前默认行为、报告内容可指定项和预留开放项。
- 技术契约和 exporter 待收敛项不再混入业务规格正文。

## 2026-05-31 文档知识库按业务特性重构

- 文档根目录统一为 `docs/`。
- 业务规格按模板管理、通用对话、报告生成、报告管理和文档导出组织。
- 报告生成明确为通用对话机制下的一种业务场景。
- 模板管理、报告管理保持纯粹的资源管理职责。
- 规格和实现变更日志按年度维护。

## 2026-05-30 BI Engine 预览与本地设计器

- 报告页面使用 BI Engine 作为统一预览和本地编辑内核。
- 对话生成过程中可以逐步预览已经生成的报告内容。
- 报告详情、对话右侧工作区和本地设计器使用同一份报告内容。
- 连续阅读型报告和分页汇报型报告都提供可导航的大纲。
- 本地设计器支持调整、重置和下载 JSON；修改暂不自动保存到后台。

## 2026-05-29 Word 与 PPT 默认样式优化

- Word 封面背景图铺满首页，报告人和时间位于首页右下角。
- Word 目录支持编号、点击跳转和适度顶部留白。
- Word 正文目录使用真正的标题层级，标题与上一段内容之间保留间距。
- Word 宽表格自动压缩列宽，空表格显示“无数据”。
- PPT 默认页脚改为 `ChatBI`，页码固定在右下角。
- PPT 封面不显示左上角页标题，普通内容页标题不再拼接报告名称前缀。
- PPT 表格使用更紧凑的默认样式，并限制在页面安全区域内。

## 2026-05-28 Word、PPT 图表与组合表格

- Word 和 PPT 可以输出折线图、柱状图和饼图等常用图表。
- 部分复杂图表在 Office 中使用兼容展示方式。
- 组合表格支持多个子表纵向紧密拼接，各子表可以拥有不同列结构，但总宽度保持一致。
- 表格表头使用浅色背景，提升阅读层次。

## 2026-05-26 章节重新生成

- 报告生成完成后，可以选择单个章节，修改分析重点并重新生成。
- 新章节先作为预览返回，不会自动覆盖原报告。
- 已经生成的下载文档不会因为预览章节而失效。

## 2026-05-14 分页汇报与结构化报告内容

- 报告支持连续阅读型和分页汇报型两种结构。
- 分页汇报可以表达封面、内容页和封底，适合 PPT 场景。
- 正式报告补充参数、大纲和生成证据，便于预览、编辑和追溯。

## 2026-05-12 动态内容扩展

- 模板可以通过外部能力生成目录、章节或分页页面内容。
- 动态内容失败时，系统返回可定位错误，不静默生成不完整报告。
- 模板可以根据用户选择展开重复目录、章节和页面。

## 2026-04-27 表格展示增强

- 普通表格支持多列合并和连续行合并。
- 组合表格可以混合查询结果和总结内容。
- 表格标题、默认展示行数和列定义进入模板展示能力。

## 2026-04-21 报告编辑上下文增强

- 报告生成后保留章节大纲、参数和生成证据，便于页面回显和后续编辑。
- 报告详情可以追溯来源会话和使用的模板。

## 2026-04-20 参数与组合表格

- 参数候选项统一展示名称、实际取值和查询表达。
- 用户回填参数时只需提交原始值。
- 模板正式支持组合表格，并为重新生成保留必要上下文。

## 2026-04-19 对话流式体验

- 报告生成过程支持持续返回进度和局部内容。
- 用户可以看到参数补充、内容生成和报告完成等阶段。
- 会话支持结构化追问和确认。

## 2026-04-18 报告系统基线

- 建立模板管理、通用对话、报告生成、报告管理和文档导出主流程。
- 模板、生成草稿、正式报告和导出文档形成完整业务链路。
- 引入用户隔离、来源追溯和统一结构校验。
