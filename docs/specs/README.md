# ReportSystem 业务规格

本目录说明 ReportSystem 面向用户提供什么能力，以及这些能力如何使用。这里不描述代码结构、数据库表、接口报文或内部实现方式。

## 业务主线

用户可以围绕一个报告主题，选择或创建模板，通过对话补充生成条件，确认报告大纲，生成正式报告，并按需要导出 Word、PPT 或 PDF 文档。

```text
维护报告模板
  -> 在对话中提出报告需求
  -> 补充条件并确认大纲
  -> 生成和预览报告
  -> 调整局部内容
  -> 导出文档
```

## 功能导航

1. [模板管理](template-management/README.md)：维护可复用的报告模板。
2. [通用对话](conversation/README.md)：管理会话和历史记录。
3. [报告生成](report-generation/README.md)：通过对话生成报告并重新生成单个章节。
4. [智能问数](data-analysis/README.md)：通过自然语言查询业务数据并获得解释和图表。
5. [报告管理](report-management/README.md)：查看、预览和管理已经生成的报告。
6. [文档导出](document-export/README.md)：生成 Word、PPT 和 PDF 文档。
7. [规格变更日志](changelog/README.md)：按年度记录用户可感知的能力变化。

## 统一原则

- 报告生成是通用对话中的一种业务场景；普通会话能力不依赖报告模板。
- 模板管理只维护可复用的模板，不在保存模板时直接生成报告。
- 报告管理只管理已经生成的报告，不承担模板识别和内容生成。
- 同一份正式报告内容用于页面预览、本地编辑和文档导出，避免不同入口看到不同结果。
- BI Engine 是页面预览和本地编辑的统一渲染内核。

## 术语映射

业务规格优先使用用户能理解的名称。研发阅读时，可以通过下表对应到内部术语。

| 业务名称 | 内部术语 | 说明 |
|---|---|---|
| 报告模板 | `ReportTemplate` | 可重复使用的报告结构和生成要求 |
| 生成草稿 | `TemplateInstance` | 某次生成过程中形成的条件、大纲和内容草稿 |
| 正式报告内容 | `Report DSL` | 供预览、编辑和导出的统一报告内容 |
| 正式报告 | `ReportInstance` | 已经保存、可以再次查看和导出的报告资源 |
| 导出文档 | `DocumentArtifact` | Word、PPT、PDF 等下载产物 |

接口、Schema 和字段级技术手册见 [实现设计中的技术契约](../implementation/contracts/README.md)。
