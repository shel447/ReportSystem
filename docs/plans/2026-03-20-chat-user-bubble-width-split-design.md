# Chat User Bubble Width Split Design

## Goal
让聊天页中的用户消息比助手消息更轻、更窄：用户消息按内容收缩显示，但最大不超过主工作区的 40%；助手消息保持当前稳定的 75% 宽度轨道。

## Decision
- 助手消息维持固定宽度：消息主体 `75%`，气泡填满主体。
- 用户消息改为内容驱动：主体 `fit-content`，最大 `40%`。
- 移动端继续统一回落为 `100%`，避免窄屏挤压。

## Impact
- 助手消息仍稳定，便于承载长解释、参数卡和大纲确认。
- 用户消息更像即时对话输入，短句不再占用过宽空间。
- 消息流整体层次更清晰，但不会再出现两边都固定一样宽导致的笨重感。
