# Chat Assistant Plain Text Width Split Design

## Goal
让助手的纯文本消息也像用户消息一样按内容收缩显示，但最大不超过主工作区的 40%；带动作面板的助手消息继续保持当前稳定宽度。

## Decision
- 用户消息保持现有规则：内容驱动，最大 40%。
- 助手纯文本消息新增一个轻量标记，只对无 action 的助手消息启用内容驱动宽度。
- 助手带面板消息保持 75% 宽度轨道，不受影响。
- 移动端继续统一回落为 100%。

## Impact
- 普通问答会更轻，用户与助手的短句风格统一。
- 参数确认、大纲确认、候选面板等复杂消息不回退。
