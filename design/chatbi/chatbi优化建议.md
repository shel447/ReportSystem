# ChatBI 优化建议

> 本文只记录建议调整点，不重复写无变化内容。

## 1. 对统一对话协议的建议

1. 明确 `ask` 与 `answer` 的业务扩展挂载点，避免不同业务在外层包络上自行加字段。
2. 明确 `reply` 的结构化回填规则，尤其是表单确认和二次编辑场景。
3. 对流式事件补充业务载荷增量约束，避免前后端对同一事件的解释不一致。

## 2. 对报告系统接入的建议

1. `REPORT` 类型应允许同时返回 `report` 与 `templateInstance`，支撑生成中展示和二次编辑。
2. 对统一对话之外的业务资源族保持独立，例如模板、报告、文档生成与下载。
3. 模板实例应保持树状主体，平铺 delta 只作为辅助视图，不应成为唯一真相。

## 3. 对动态参数的建议

1. 动态参数候选项数据源协议要统一，不应每个模板单独描述请求方法和响应体。
2. 三元组 `display / value / query` 语义要贯穿参数定义、运行态、外部数据源和最终执行层。
