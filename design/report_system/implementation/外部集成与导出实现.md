# 外部集成与导出实现

## 1. LLM 集成

LLM 能力只通过统一网关使用，应用层只依赖以下能力 port：

- 模板匹配补充判断
- 参数提取
- 诉求实例化文案整理
- 章节内容生成
- 章节总结和报告汇总

## 2. 动态参数数据源

模板只声明 `source`。系统负责：

1. 将 `contextValues` 组装成标准请求体
2. 向外部接口发送 `POST`
3. 校验外部响应满足 `parameter-option-source-response.schema.json`
4. 返回 `options + defaultValue`

## 3. 查询与执行

数据查询基础设施负责：

- 解释 `runtimeContext.bindings`
- 运行 SQL/API 查询
- 返回结构化结果和执行证据

运行时禁止提前把多值参数拼成展示文本再送执行层；执行层消费的是三元组数组和最终 `resolvedQuery`。

## 4. Java 导出器

导出链路固定为：

- `Report DSL -> JavaOfficeExporterGateway -> word/ppt`
- `word/ppt -> PdfConverterGateway -> pdf`

导出请求和响应结构必须与设计包 `06-文档生成与导出架构.md` 一致。

当前落地位置：

- Java 服务源码：`services/java-office-exporter/src/report/system/exporter/JavaOfficeExporterServer.java`
- 构建脚本：`services/java-office-exporter/build.ps1`
- 运行脚本：`services/java-office-exporter/run.ps1`
- 运行时日志：`output/runtime/java-office-exporter.{out,err}.log`

当前 Python 后端通过 `src/backend/infrastructure/exporter/java_office.py` 以 HTTP 方式调用该服务，并在本地未启动时自动编译并拉起服务。
