# 外部集成与导出实现

## 1. LLM 集成

LLM 能力只通过统一网关使用，应用层只依赖以下能力 port：

- 模板匹配补充判断
- 参数提取
- 诉求实例化文案整理
- 章节内容生成
- 章节总结和报告汇总

## 2. 动态参数数据源

模板只声明 `source`。系统负责：

1. 将 `contextValues` 组装成标准请求体
2. 向外部接口发送 `POST`
3. 校验外部响应满足 `parameter-option-source-response.schema.json`
4. 返回 `options + defaultValue`

## 3. 查询与执行

数据查询基础设施负责：

- 解释 `runtimeContext.bindings`
- 运行 SQL/API 查询
- 返回结构化结果和执行证据

运行时禁止提前把多值参数拼成展示文本再送执行层；执行层消费的是三元组数组和最终 `resolvedQuery`。

## 4. Java 导出器

导出链路固定为：

- `Report DSL -> JavaOfficeExporterGateway -> word/ppt`
- `word/ppt -> PdfConverterGateway -> pdf`

导出请求和响应结构必须与设计包 `06-文档生成与导出架构.md` 一致。

### 4.1 技术栈

- Java 21 + Maven 构建
- Apache POI 5.4.0 (poi-ooxml) 生成 DOCX/PPTX
- Jackson 2.18.2 反序列化 Report DSL JSON
- JDK 内置 `com.sun.net.httpserver` 提供 HTTP 服务
- maven-shade-plugin 打包 fat jar

### 4.2 代码分层

参照 poi-dsl-exporter 架构设计，Java 导出器采用分层结构：

- `core/` — 导出编排 (`ExporterOrchestrator`)、导出器接口 (`DocumentExporter`)、请求模型 (`ExportRequest`)、目标枚举 (`ExportTarget`)、DSL 校验 (`DslValidator`)
- `model/` — Report DSL JSON 模型 (Jackson 反序列化)：`ExportPayload`, `ReportDslModel`, `ReportBasicInfo`, `ReportCatalog`, `ReportSection`, `ReportCover`, `ReportSignaturePage`, `ReportSummary`, `BackCoverConfig`
- `style/` — 主题令牌 (`ThemeTokens`) 与样式解析 (`StyleResolver`)，内置 enterprise-light/enterprise-dark 两套主题
- `chart/` — 图表规范 (`ChartSpec`) 与解析器 (`ChartSpecParser`)，从 ChartComponent 提取 categories/series，支持 line/bar/pie 原生渲染 + 其他类型降级为数据表格
- `docx/` — DOCX 导出主流程 (`ReportDocxExporter`) + 渲染器：封面/签署页 (`DocxCoverRenderer`)、文本/Markdown (`DocxTextRenderer`)、表格含合并 (`DocxTableRenderer`)、图表含 XDDF 原生图表 (`DocxChartRenderer`)
- `pptx/` — PPTX 导出主流程 (`ReportPptxExporter`) + 渲染器：表格 (`PptxTableRenderer`)、图表 (`PptxChartRenderer`)，支持 flow 和 paged 两种结构
- `HttpServerMain.java` — HTTP 入口，路由 `/health` 和 `/exports/{word|ppt}`

### 4.3 DOCX 导出流程

1. 创建 `XWPFDocument`，设置页边距 (14mm)
2. 渲染封面页 (标题/副标题/作者/日期/封面内容项) + 分页
3. 渲染目录页 (catalogs 层级列表) + 分页
4. 递归遍历 catalogs → sections → components：
   - 目录标题 → Heading1/2/3
   - 文本/Markdown → 段落 (支持粗体/斜体/列表)
   - 表格 → XWPFTable (表头着色、交替行背景、列/行合并)
   - 图表 → XDDF 原生图表 (line/bar/pie) 或降级为数据表格
   - 复合表 → 遍历子表逐个渲染
5. 渲染报告摘要
6. 渲染签署页
7. 设置页眉/页脚 (来自 basicInfo.header/footer)

### 4.4 PPTX 导出流程

1. 创建 `XMLSlideShow`，设置 16:9 尺寸
2. 封面幻灯片 (标题居中 + 副标题 + 作者/日期)
3. 目录幻灯片 (catalogs 编号列表)
4. flow 模式：每个 catalog 生成章节标题页，每个 section 生成内容页
5. paged 模式：按 content[] 中的 Slide/SlideSection 逐页生成
6. 每页内容按组件类型渲染：文本框、XSLFTable、图表信息卡
7. 摘要幻灯片 + 封底

### 4.5 图表策略

| 原始类型 | 渲染方式 | 说明 |
|---|---|---|
| line | 原生 XDDF 线图 | DOCX 支持 |
| bar | 原生 XDDF 柱图 | DOCX 支持 |
| pie | 原生 XDDF 饼图 | DOCX 支持 |
| scatter/radar/heatmap/kline 等 | 降级为数据表格 | 附带图表类型说明 |

### 4.6 文件位置

- Java 服务源码：`services/java-office-exporter/src/main/java/report/system/exporter/`
- Maven 配置：`services/java-office-exporter/pom.xml`
- 构建脚本：`services/java-office-exporter/build.sh` / `build.ps1`
- 运行脚本：`services/java-office-exporter/run.sh` / `run.ps1`
- 产物 JAR：`services/java-office-exporter/target/java-office-exporter-0.1.0.jar`
- 运行时日志：`output/runtime/java-office-exporter.{out,err}.log`

### 4.7 Python 集成

Python 后端通过 `src/backend/infrastructure/exporter/java_office.py` 以 HTTP 方式调用该服务：

- 自动检测 JAR 是否过期（比较源码 mtime 与 JAR mtime）
- 自动执行 `mvn clean package -q` 构建
- 自动启动 `java -jar` 进程
- 健康检查 `/health`，30 秒超时
