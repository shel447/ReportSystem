# 外部集成与导出实现

## 1. LLM 集成

LLM 能力只通过统一网关使用，应用层只依赖以下能力 port：

- 模板匹配补充判断
- 参数提取
- 诉求实例化文案整理
- 章节内容生成
- 章节总结和报告汇总

## 2. 动态参数数据源

模板只声明 `source`。系统负责：

1. 将 `contextValues` 组装成标准请求体
2. 向外部接口发送 `POST`
3. 校验外部响应满足 `parameter-option-source-response.schema.json`
4. 返回 `options + defaultValue`

## 3. 查询与执行

数据查询基础设施负责：

- 解释 `runtimeContext.bindings`
- 运行 SQL/API 查询
- 返回结构化结果和执行证据

运行时禁止提前把多值参数拼成展示文本再送执行层；执行层消费的是三元组数组和最终 `resolvedQuery`。

## 4. Java 导出器

Java 导出器源码对齐 `chat_bi_ui/tools/poi-dsl-exporter/src/main/java/com/chatbi`，当前模块提供 CLI/库式导出能力：

- `VDoc/BI Engine DSL JSON -> com.chatbi.exporter.CliMain -> docx/pptx`
- `word/ppt -> PdfConverterGateway -> pdf` 仍由后续集成层衔接

说明：旧版 `com.bi.report.generation.HttpServerMain` HTTP 服务入口已移除；如 Python 后端仍需要 `/health`、`/exports/{word|ppt}` 协议，应在后续集成层新增适配器，而不是在本次照搬源码中保留旧实现。

### 4.1 技术栈

- Java 21 + Maven 构建
- Apache POI 5.4.0 (poi-ooxml) 生成 DOCX/PPTX
- Jackson 2.18.2 反序列化 VDoc/BI Engine DSL JSON
- maven-shade-plugin 打包 fat jar

### 4.2 代码分层

Java 导出器采用 `com.chatbi` 源码树：

- `com.chatbi.exporter.CliMain` — CLI 入口，解析 `--input/--output/--target/--theme/--strict`
- `com.chatbi.exporter.core` — 导出编排、导出器接口、目标枚举和 DSL 校验
- `com.chatbi.exporter.model` — VDoc/VNode 运行时模型
- `com.chatbi.exporter.docx` — DOCX 导出主流程
- `com.chatbi.exporter.pptx` — PPTX 导出主流程
- `com.chatbi.exporter.chart` — 图表规范、数据集解析和 POI 原生图表渲染
- `com.chatbi.exporter.table` — 表格模型与解析
- `com.chatbi.exporter.style` — 主题令牌与样式解析
- `com.chatbi.exporter.util` — DSL 读取与 BI Engine DSL 归一化
- `com.chatbi.report.dsl` — BI Engine Report DSL Java 模型

`com.chatbi.report.dsl` 已支持独立 Jackson JSON round-trip，可通过 `ReportDslJson` 读取和写出 `Report` 模型。当前导出运行时仍使用 `DslReader -> VDoc/Normalizer` 链路，不直接消费 `ReportDslJson`。

### 4.3 DOCX 导出流程

1. 创建 `XWPFDocument`，设置页边距 (14mm)
2. 渲染封面页 (标题/副标题/作者/日期/封面内容项) + 分页
3. 渲染目录页 (catalogs 层级列表) + 分页
4. 递归遍历 catalogs → sections → components：
   - 目录标题 → Heading1/2/3
   - 文本/Markdown → 段落 (支持粗体/斜体/列表)
   - 表格 → XWPFTable (表头着色、交替行背景、列/行合并)
   - 图表 → XDDF 原生图表 (line/bar/pie) 或降级为数据表格
   - 复合表 → 遍历子表逐个渲染
5. 渲染报告摘要
6. 渲染签署页
7. 设置页眉/页脚 (来自 basicInfo.header/footer)

### 4.4 PPTX 导出流程

1. 创建 `XMLSlideShow`，设置 16:9 尺寸
2. 封面幻灯片 (标题居中 + 副标题 + 作者/日期)
3. 目录幻灯片 (catalogs 编号列表)
4. flow 模式：每个 catalog 生成章节标题页，每个 section 生成内容页
5. paged 模式：按 content[] 中的 Slide/SlideSection 逐页生成
6. 每页内容按组件类型渲染：文本框、XSLFTable、图表信息卡
7. 摘要幻灯片 + 封底

### 4.5 图表策略

| 原始类型 | 渲染方式 | 说明 |
|---|---|---|
| line | 原生 XDDF 线图 | DOCX 支持 |
| bar | 原生 XDDF 柱图 | DOCX 支持 |
| pie | 原生 XDDF 饼图 | DOCX 支持 |
| scatter/radar/heatmap/kline 等 | 降级为数据表格 | 附带图表类型说明 |

### 4.6 文件位置

- Java 导出器源码：`services/java-office-exporter/src/main/java/com/chatbi/`
- Maven 配置：`services/java-office-exporter/pom.xml`
- 构建脚本：`services/java-office-exporter/build.sh` / `build.ps1`
- 运行脚本：`services/java-office-exporter/run.sh` / `run.ps1`
- 产物 JAR：`services/java-office-exporter/target/java-office-exporter-0.1.0.jar`

### 4.7 Python 集成

Python 后端当前仍通过 `src/backend/infrastructure/exporter/java_office.py` 以 HTTP 方式调用旧 Java 服务协议：

- 自动检测 JAR 是否过期（比较源码 mtime 与 JAR mtime）
- 自动执行 `mvn clean package -q` 构建
- 自动启动 `java -jar` 进程
- 健康检查 `/health`，30 秒超时

本次 Java 源码照搬后，Java JAR 默认只提供 CLI 入口；后端 HTTP 集成需要后续新增适配层或改造为进程式 CLI 调用。
