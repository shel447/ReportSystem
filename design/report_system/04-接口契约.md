# 04. 接口契约

## 1. 公开接口面

```text
POST   /rest/chatbi/v1/templates
GET    /rest/chatbi/v1/templates
GET    /rest/chatbi/v1/templates/{id}
PUT    /rest/chatbi/v1/templates/{id}
DELETE /rest/chatbi/v1/templates/{id}
POST   /rest/chatbi/v1/templates/import/preview
GET    /rest/chatbi/v1/templates/{id}/export

GET    /rest/chatbi/v1/chat
POST   /rest/chatbi/v1/chat
GET    /rest/chatbi/v1/chat/{conversationId}
DELETE /rest/chatbi/v1/chat/{conversationId}
POST   /rest/chatbi/v1/chat/forks

GET    /rest/chatbi/v1/reports/{reportId}
POST   /rest/chatbi/v1/reports/{reportId}/document-generations
GET    /rest/chatbi/v1/reports/{reportId}/documents/{documentId}/download

POST   /rest/chatbi/v1/parameter-options/resolve
```

统一身份头：

```http
X-User-Id: <external-user-id>
```

## 2. 模板接口

### 2.1 模板详情

模板接口直接返回正式模板对象：`ReportTemplate`

```json
{
  "id": "tpl_network_daily",
  "category": "network_operations",
  "name": "网络运行日报",
  "description": "面向网络运维中心的统一日报模板。",
  "schemaVersion": "template.v3",
  "parameters": [],
  "catalogs": []
}
```

### 2.2 导入预览

请求：

```json
{
  "content": "...原始模板内容..."
}
```

响应：

```json
{
  "normalizedTemplate": {},
  "warnings": []
}
```

## 3. `/chat` 请求契约

### 3.1 ChatRequest

```json
{
  "conversationId": "conv_001",
  "chatId": "chat_001",
  "question": "帮我生成总部网络运行日报",
  "instruction": "generate_report",
  "reply": null,
  "attachments": [],
  "histories": [],
  "requestId": "req_001",
  "apiVersion": "v1"
}
```

关键字段：

- `conversationId`：会话 id
- `chatId`：当前轮 id
- `instruction`：当前正式支持 `generate_report`、`extract_report_template`
- `reply`：承接参数回填、确认生成等结构化回复

### 3.2 Reply

```json
{
  "type": "confirm_params",
  "parameters": {
    "report_date": [
      {
        "display": "2026-04-18",
        "value": "2026-04-18",
        "query": "2026-04-18"
      }
    ]
  },
  "reportContext": {
    "templateInstance": {}
  }
}
```

规则：

- `fill_params` 用于参数补齐
- `confirm_params` 用于确认生成
- 诉求确认时必须携带完整 `templateInstance`，以支持后台做最终骨架判定和生成输入冻结

## 4. `/chat` 响应契约

### 4.1 ChatResponse

```json
{
  "conversationId": "conv_001",
  "chatId": "chat_001",
  "status": "waiting_user",
  "steps": [],
  "ask": {},
  "answer": null,
  "errors": [],
  "requestId": "req_001",
  "timestamp": 1713427200000,
  "apiVersion": "v1"
}
```

### 4.2 Ask

```json
{
  "mode": "form",
  "type": "confirm_params",
  "title": "请确认报告诉求",
  "text": "请确认报告诉求后开始生成。",
  "parameters": [],
  "reportContext": {
    "templateInstance": {}
  }
}
```

规则：

- 模板选择通过参数实现，不再单独设计模板选择阶段协议
- 诉求确认通过 `reportContext.templateInstance` 返回完整模板实例

### 4.3 Answer

```json
{
  "answerType": "REPORT",
  "answer": {
    "reportId": "rpt_001",
    "status": "generating",
    "report": {},
    "templateInstance": {},
    "documents": [],
    "generationProgress": {
      "totalSections": 8,
      "completedSections": 2
    }
  }
}
```

规则：

- `/chat` 在确认参数后返回的是“生成中的报告”
- `report` 是正式 `Report DSL`
- `templateInstance` 一并返回，支撑后续二次编辑后重新生成

### 4.4 ChatStreamEvent

```json
{
  "conversationId": "conv_001",
  "chatId": "chat_003",
  "eventType": "answer",
  "sequence": 6,
  "timestamp": 1713427200300,
  "answer": {
    "answerType": "REPORT",
    "answer": {
      "reportId": "rpt_001",
      "status": "generating",
      "report": {},
      "templateInstance": {},
      "documents": []
    }
  }
}
```

事件类型：

- `status`
- `step_delta`
- `ask`
- `answer`
- `error`
- `done`

## 5. `/reports` 契约

### 5.1 获取报告详情

```json
{
  "reportId": "rpt_001",
  "status": "available",
  "answerType": "REPORT",
  "answer": {
    "reportId": "rpt_001",
    "status": "available",
    "report": {},
    "templateInstance": {},
    "documents": []
  }
}
```

规则：

- 这是一次性返回的完成态报告
- `answer` 必须与 `/chat` 中 `REPORT.answer.answer` 结构等价
- 与流式生成时的差异只在于：这里是完整完成态

### 5.2 文档生成

请求：

```json
{
  "formats": ["word", "ppt", "pdf"],
  "pdfSource": "word",
  "theme": "default",
  "strictValidation": true,
  "regenerateIfExists": false
}
```

响应：

```json
{
  "reportId": "rpt_001",
  "jobs": [
    {
      "jobId": "job_word_001",
      "format": "word",
      "status": "queued"
    },
    {
      "jobId": "job_pdf_001",
      "format": "pdf",
      "status": "blocked_by_dependency",
      "dependsOn": "job_word_001"
    }
  ]
}
```

## 6. `/parameter-options/resolve` 契约

请求：

```json
{
  "parameterId": "scope",
  "openSource": {
    "url": "https://example.internal/api/network/scopes/options"
  },
  "contextValues": {
    "report_date": [
      {
        "display": "2026-04-18",
        "value": "2026-04-18",
        "query": "2026-04-18"
      }
    ]
  }
}
```

响应：

```json
{
  "options": [],
  "defaultValue": []
}
```

规则：

- 服务端用 `contextValues` 组装对外部 `openSource.url` 的统一请求体
- 外部接口的正式请求、响应结构由本设计包的 JSON Schema 统一定义
