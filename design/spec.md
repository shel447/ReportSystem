## 1. 产品概述

### 1.1 产品定位

智能报告系统当前是一个以**报告生成**为主能力的统一智能助手，同时集成：

- 智能问数
- 智能故障
- 定时任务自动生成
- Markdown 文档导出

### 1.2 核心价值

| 价值维度 | 说明 |
|---------|------|
| 统一入口 | 同一对话入口承接报告、问数、故障三类任务 |
| 结构化确认 | 参数确认、大纲确认、生成基线三段式保证生成链路可追踪 |
| 双层模板 | 模板同时维护用户蓝图与系统执行链路 |
| 任务复用 | 报告实例可更新、可 fork，会话可恢复与分支 |
| 调度自动化 | 可从既有实例创建定时任务，并映射业务报告时间 |

## 2. 当前已实现规格

### 2.1 系统设置

- 支持 Completion / Embedding 全局配置
- 支持连接测试
- 支持模板语义索引重建
- 未配置系统设置时，对话生成与真实调用链路会明确报错，不再回退 mock

### 2.2 报告模板

- 创建、编辑、删除、克隆、导出单模板 JSON
- 模板主结构采用 `parameters / sections`
- 参数支持：
  - `free_text / date / enum / dynamic`
  - `interaction_mode = form | chat`
- 章节支持双层模型：
  - `outline.document + outline.blocks[]`
  - `content.datasets + presentation`
- 模板工作台支持：
  - 参数工作台
  - 章节树
  - 蓝图 / 执行链路 / 同步状态页签
  - 蓝图预览 / 执行预览 / 模板 JSON

### 2.3 统一对话模块

- `/chat` 默认空态，不自动恢复最近会话，也不预创建会话
- 首条真实用户消息发送后才创建会话，并以该消息生成标题
- 支持三类一级能力：
  - `report_generation`
  - `smart_query`
  - `fault_diagnosis`
- 采用单活任务模型：
  - 一个会话同一时刻只有一个 `active_task`
  - 任务切换需要确认
- 支持会话历史：
  - 新建会话
  - 切换会话
  - 删除会话
  - 折叠会话栏

### 2.4 报告生成对话

- 模板匹配
- 参数收集
- 参数确认
- 大纲确认
- 生成报告实例
- 生成 Markdown 文档

参数收集当前支持混合模式：

- `form` 参数：返回 `ask_param`
- `chat` 参数：返回自然语言追问文本
- 同一模板内按参数顺序混排

大纲确认当前支持：

- 实例级蓝图树展示
- 参数片段 inline 编辑
- 非参数文本编辑后节点退化为普通句子
- `foreach` 展开
- 结构化蓝图值注入执行链路

### 2.5 报告实例

- 列表、详情、删除
- 章节重生成
- 查看确认大纲 / 生成基线
- 从实例发起“更新”会话
- 从实例来源消息发起 “Fork” 会话
- 实例当前额外支持：
  - `report_time`
  - `report_time_source`
  - `has_generation_baseline`
  - `supports_update_chat`
  - `supports_fork_chat`

### 2.6 报告文档

- 生成 Markdown 文档
- 下载 Markdown 文档
- 文档记录列表
- 文档与实例关联查看

### 2.7 定时任务

- 定时任务列表、创建、更新、删除、暂停、恢复、立即执行
- 当前创建入口固定为“从已有报告实例创建”
- 支持：
  - `schedule_type`
  - `cron_expression`
  - `auto_generate_doc`
  - `time_param_name`
  - `time_format`
  - `use_schedule_time_as_report_time`
- 执行时采用双时间模型：
  - `created_at` 保留真实执行时间
  - `report_time` 表达业务报告时间

### 2.8 智能问数与智能故障

- 智能问数：
  - 统一由对话模块路由进入
  - 返回结构化摘要和查询调试上下文
- 智能故障：
  - 返回故障现象、初步判断、风险等级、原因、建议
- 两者都支持在报告流程中显式切换

## 3. 当前约束与行为边界

### 3.1 报告生成

- 所有 `required` 参数都必须显式确认
- 大纲确认统一发生在实例生成之前
- 当前只输出 Markdown，不输出 PDF
- 旧模板结构仍可兼容读取，但保存后统一按 `v2.0` 结构维护

### 3.2 统一对话模块

- v1 不支持任务栈
- 问数后不会自动回到之前的报告任务
- 当 `chat` 模式参数待收集时，普通自然语言优先作为参数答案
- 只有显式切换意图才允许中断当前报告流程

### 3.3 定时任务

- 当前不支持脱离源报告实例直接创建
- `run-now` 场景下 `scheduled_time = actual_run_time`
- 失败不自动重试
