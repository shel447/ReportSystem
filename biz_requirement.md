## 一、业务流程设计

### 1、准备阶段

1.1、注册报告模板

### 2、运行阶段

2.1、用户访问对话（chat）接口，输入问题

2.2、系统接收到用户问题，根据用户问题匹配报告模板

2.3、根据报告模板的配置内容，提示用户补充必要与可选信息

2.4、用户确认后，系统根据报告模板与用户补充信息，生成报告实例，并在对话接口返回给用户预览、修改

2.5、用户修改后，可以对局部内容重新生成，更新报告实例

2.6、当用户最终确认内容无误后，生成报告文档

2.7、用户可以通过报告文档管理接口，对报告文档进行增删查改或者下载

## 二、关键概念

| 概念     | 定义                                                         | 补充说明 |
| -------- | ------------------------------------------------------------ | -------- |
| 报告模板 | 用户预先定义的报告模板。包括报告类型、报告场景、报告内容参数、报告大纲 |          |
| 报告实例 | 在报告模板的基础上，填充报告内容参数后，使用大语言模型技术栈生成报告实例。<br>报告实例是一种中间态，既能看到最终呈现样式，又保留着对报告模板具体内容以及参数的引用信息 |          |
| 报告文档 | 报告实例的物理载体，文档类型可以是word、pdf                  |          |
